# 데이터 인벤토리

## 1. 문서 목적
MOSAIC 프로젝트에서 사용할 공개데이터의 출처, 분석 단위, 갱신 주기, 기간, 결합키 및 주요 위험을 관리한다.

원천 데이터 파일 자체는 Git에 커밋하지 않으며, 데이터의 출처와 메타데이터만 저장소에서 관리한다.

---

## 2. 최초 분석 범위
- 공간 범위: 서울특별시
- 시간 범위: 2021년 1분기~2025년 4분기
- 주요 분석 단위: 상권 x 서비스 업종 x 연도.분기
- 최초 목표변수 후보: 상권.업종 단위 점포당 추정매출 대리변수

목표변수 후보는 다음과 같이 정의한다.
```
점포당 추정매출 대리변수
= 상권.업종별 추정매출 / 상권.업종별 점포 수
```

이 값은 실제 특정 점포의 매출이 아니다.

## 3. 핵심 데이터셋

### DATA-001: 서울시 상권분석서비스 추정매출-상권
- 제공 기관: 서울신용보증재단
- 출처 시스템: 서울시 상권분석서비스
- 데이터 식별자: OA-15572
- 분석 역할: 목표변수 생성
- 시간 단위: 분기
- 공간 단위: 상권
- 업종 단위: 서비스 업종
- 금액 단위: 원
- 최초 사용 기간: 2021~2025년
- 저장 위치: `data/raw/seoul_commercial/sales/`
- 예상 결합 키:
  - 기준 연도.분기 코드
  - 상권 구분 코드
  - 상권 코드
  - 서비스 업종 코드
- 주요 위험:
  - 추정매출은 실제 개별 점포 매출이 아니다.
  - 2024년부터 상권 공간 단위가 변경됐다.
  - 연도별 스키마와 컬럼명이 동일한지 검증해야 한다.
  - 점포 데이터와 결합할 때 분기와 업종 코드의 일치 여부를 검증해야 한다.

### DATA-002: 서울시 상권분석서비스 점포-상권
- 제공 기관: 서울신용보증재단
- 출처 시스템: 서울시 상권분석서비스
- 데이터 식별자: OA-15577
- 분석 역할:
  - 점포 수 산출
  - 경쟁 강도 변수 생성
  - 개업.폐업 특성 생성
  - 프랜차이즈 비중 생성
- 시간 단위: 연도.분기 여부를 원본 스키마에서 재확인
- 공간 단위: 상권
- 업종 단위: 서비스 업종
- 최초 사용 기간: 2021~2025년
- 저장 위치: `data/raw/seoul_commercial/stores/`
- 예상 결합 키:
  - 기준 연도.분기 코드
  - 상권 구분 코드
  - 상권 코드
  - 서비스 업종 코드
- 주요 위험:
  - 점포 수가 0인 행은 점포당 매출 계산에 사용할 수 없다.
  - 전체 점포 수와 일반.프랜차이즈 점포 수의 합이 일치하는지 검사해야 한다.
  - 폐업률과 개업률의 분모 정의를 원본 데이터 설명에서 확인해야 한다.
  - 추정매출 데이터와 갱신 시점이 다를 수 있다.

### DATA-003: 서울시 상권분석서비스 영역-상권
- 제공 기관: 서울신용보증재단
- 출처 시스템: 서울시 상권분석서비스
- 데이터 식별자: OA-15560
- 분석 역할:
  - 상권 경계 지도 시각화
  - 상권 중심점 계산
  - 인접 상권 관계 생성
  - 공간 거리 및 공간 지연 변수 생성
- 공간 단위: 상권
- 좌표계: EPSG:5181
- 저장 위치: `data/raw/seoul_commercial/areas/`
- 예상 결합 키:
  - 상권 구분 코드
  - 상권 코드
- 주요 위험:
  - 모델이나 지도에서 사용하기 전에 좌표계 변환이 필요할 수 있다.
  - 매출.점포 데이터와 상권 코드 체계가 일치하는지 확인해야 한다.
  - 다중 폴리곤이나 유효하지 않은 geometry가 존재하는지 검사해야 한다.
  - 2024년 공간 단위 변경 전후의 경계 일관성을 확인해야 한다.

## 4. 후속 데이터 후보

### 생활인구 - 상권
예상 활용:
- 시간대별 유동 수요
- 성별.연령별 인구 구조
- 주중.주말 상권 특성

도입 시점:
핵심 세 데이터의 결합과 품질검사가 완료된 후 추가한다.

### 직장인구 - 상권
예상 활용:
- 오피스형 상권 판별
- 주간 소비수요 대리변수

주의사항:
연간 갱신 특성과 동일 값 반복 여부를 확인해야 한다.

### 지하철 승하차 인원
예상 활용:
- 역세권 접근성
- 상권 주변 교통량
- 출퇴근형 상권 구분

주의사항:
역명 표준화와 상권 경계까지의 거리 계산이 필요하다.

### 전국 상가업소 정보
예상 활용:
- 경쟁 점포 위치
- 업종 다양성
- 반경별 점포 밀도

주의사항:
서울시 상권분석서비스의 업종 코드와 분류체계가 다를 수 있다.

## 5. 데이터 수집 원칙
1. 원천 ZIP 파일은 다운로드 후 수정하지 않는다.
2. 압축 해제 및 변환은 코드로 수행한다.
3. 원본 컬럼명은 raw 단계에서 변경하지 않는다.
4. 컬럼명 표준화는 interim 단계부터 수행한다.
5. raw, interim, processed 데이터를 명확히 분리한다.
6. 결측치나 이상치를 임의로 삭제하지 않고 처리 근거를 기록한다.
7. 데이터 정의 변경을 발견하면 의사결정 로그에 기록한다.

## 6. 최초 품질검사 항목

### 파일 수준
- ZIP 파일 손상 여부
- 파일 수
- 파일 크기
- 문자 인코딩
- CSV 구분자
- SHA-256 체크섬

### 스키마 수준
- 연도별 컬럼 목록
- 컬럼 데이터 타입
- 필수 키 누락
- 중복 키
- 컬럼 추가.삭제
- 코드 길이 변화

### 값 수준
- 매출액 음수 여부
- 점포 수 음수 여부
- 점포 수 0인 행
- 매출액 0인 행
- 전체 점포 수 합계 검증
- 분기 연속성
- 상권 코드 매칭률
- 업종 코드 매칭률

### 공간 수준
- 좌표계
- 유효하지 않은 geometry
- 빈 geometry
- 중복 상권 코드
- 경계 중첩
- 중심점이 서울 범위 안에 존재하는지 여부
