# MOSAIC: 도시형 리테일 의사결정 시스템
MOSAIC은 공개 데이터를 이용해 도시형 소매 점포의 입지 선정, 예상매출 예측, 상품 수요예측 및 재고 의사결정을 지원하는 End-to-End 데이터사이언스 프로젝트다.

## 프로젝트 목표
이 프로젝트는 단순히 예측 정확도가 높은 모델을 만드는 데서 끝나지 않는다.

다음 의사결정 흐름을 구현하는 것을 모표로 한다.
```
공개데이터 수집
-> 데이터 품질 검증
-> 매출.수요 예측
-> 출점.발주 의사결정
-> 비용 및 위험 평가
-> 모델 운영과 모니터링
```

## 핵심 비즈니스 질문
1. 어느 상권을 신규 점포 후보로 검토해야 하는가?
2. 후보 상권에서 기대할 수 있는 매출 범위는 얼마인가?
3. 경쟁 점포와 예측 불확실성을 고려하면 어느 후보가 가장 적절한가?
4. 점포와 상품별 미래 수요는 얼마인가?
5. 품절과 과잉재고 비용을 줄이려면 얼마나 발주해야 하는가?

## 프로젝트 구성

### 점포개발 트랙
서울 상권, 인구, 교통, 점포 및 기상 데이터를 활용한다.

주요 과제:
- 상권 특성 분석
- 상권.업종 단위 예상매출 예측
- 신규 출점 후보 랭킹
- 경쟁 강도 및 자기잠식 위험 평가
- 공간적 일반화 성능 검증

### 점포운영 트랙
공개 소매 판매 데이터를 활용한다.

주요 과제:
- 점포.상품별 수요예측
- 계층적 시계열 예측
- 품절 및 과잉재고 비굑 계산
- 발주 정책 시뮬레이션
- 예측 모델과 재고 정책 비교

### 공통 플랫폼
- 재현 가능한 데이터 파이프라인
- 자동 테스트와 데이터 계약
- MLflow 기반 실행 관리
- Docker 기반 실행환경
- AWS 호환 학습.배포 구조
- 데이터 및 모델 드리프트 모니터링

## 데이터 해석 시 주의사항
공개 상권 데이터의 추정매출은 특정 기업이나 특정 점포의 실제 매출이 아니다.
따라서 프로젝트에서는 이를 다음과 같이 표현한다.

> 상권.업종 단위 점포당 추정매출 대리변수

또한 국내 상권 데이터와 공개 해외 소매 판매 데이터는 서로 다른 출처에서 생성된 데이터다. 
두 데이터가 동일한 기업이나 동일한 점포 운영과정에서 생성된 것처럼 해석하지 않는다.
